Work Completed

We have implemented the sequential code of the algorithm we intend to parallelize.

We have made sure that all all the libraries we intend to use, most importantly PCL+dependencies, PCLVisualizer+X11 forwarding, OpenMP can be compiled and run in our development environment.

Progress wrt original deliverables

We are about one week behind the goals and deliverables we stated in our proposal. We underestimated the difficulty of setting up our development environment. We encountered numerous issues getting Point Cloud Library and its dependencies to compile on our computers, which was the initial plan, and on the Gates cluster, which was back up plan. We’ve since decided to do the work on an Ubuntu Virtual Machine on the Google Cloud Platform, where we’re gotten the libraries to work successfully.

We should still be able to produce all our deliverables if we work harder from this point on. Our goals for the parallelism competition remain the same - a correct algorithm with at least a 3x speedup. It will be nice if we can achieve the speedup required to run in real-time. If we are able to get the algorithm fast enough to run in real time, we hope to show that at the parallelism competition. If we aren’t, then we will probably just show a sped-up video of it performing SLAM.

Concerns

We are unsure about using vector instructions to parallelize over points, because the OpenMP library might not be compatible with the ISPC compiler. We are looking at upgrading our current version of GCC to use OpenMP 4.0, which supports vector instructions, but this might not work because of PCL compatibility issues.
